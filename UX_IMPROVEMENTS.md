## ✅ 구현된 개선 사항

### 1. Pricing 페이지: 로그인 필요 안내

**위치**: `modules/pricing/PlanSelectStep.tsx`

**개선 내용**:
- 로그아웃 상태에서 "구독하기" 버튼 클릭 시
- 바로 리다이렉트하는 대신 **확인 메시지 표시**
- 사용자에게 명확한 안내 제공

**메시지 내용**:
```
구독하려면 로그인이 필요합니다.

로그인 페이지로 이동하시겠습니까?
(로그인 후 이 페이지로 다시 돌아옵니다)
```

**코드**:
```typescript
const handleSubscribe = (plan: PricingPlan) => {
  if (!isAuthenticated) {
    const confirmed = window.confirm(
      "구독하려면 로그인이 필요합니다.\n\n로그인 페이지로 이동하시겠습니까?\n(로그인 후 이 페이지로 다시 돌아옵니다)"
    );
    
    if (confirmed) {
      onLogin(); // 메인 서비스 로그인 페이지로 이동
    }
    return;
  }
  
  onSubscribe(plan); // 로그인된 경우 결제 진행
};
```

**사용자 플로우**:
```
1. 로그아웃 상태로 /pricing 접속
   ↓
2. 플랜 선택 후 "구독하기" 클릭
   ↓
3. ✨ 확인 메시지 표시
   "구독하려면 로그인이 필요합니다..."
   ↓
4. 사용자가 "확인" 클릭
   ↓
5. 메인 서비스 로그인 페이지로 이동
   (returnUrl=/pricing 포함)
   ↓
6. 로그인 완료 후 /pricing으로 자동 복귀
   ↓
7. 다시 "구독하기" 클릭 → 결제 진행
```

**취소 시 동작**:
```
1. "구독하기" 클릭
   ↓
2. 확인 메시지 표시
   ↓
3. 사용자가 "취소" 클릭
   ↓
4. 현재 페이지에 그대로 유지 ✅
```

## 🎯 UX 개선 이유

### Before (개선 전)
```
❌ 사용자가 "구독하기" 클릭
    → 갑자기 다른 사이트(메인 서비스)로 이동
    → 사용자 혼란: "왜 다른 곳으로 갔지?"
```

### After (개선 후)
```
✅ 사용자가 "구독하기" 클릭
    → "로그인이 필요합니다" 메시지 표시
    → 사용자 이해: "아, 로그인해야 하는구나"
    → 사용자 선택: 확인 or 취소
    → 확인 시 메인 서비스로 이동
```

## 🔮 추가 개선 고려 사항

### 1. 모달 UI로 업그레이드 (선택사항)

**현재**: `window.confirm()` 사용 (브라우저 기본 알림)

**개선안**: 커스텀 모달 컴포넌트
```tsx
<LoginRequiredModal
  isOpen={showModal}
  onConfirm={handleLogin}
  onCancel={() => setShowModal(false)}
  message="구독하려면 로그인이 필요합니다."
  returnTo="이 페이지"
/>
```

**장점**:
- 브랜드 일관성 유지
- 더 나은 시각적 표현
- 추가 정보 제공 가능 (혜택 등)

**단점**:
- 추가 구현 필요
- 모달 컴포넌트 관리 필요

### 2. Toast 알림 추가 (선택사항)

로그인 후 복귀 시:
```
✅ 로그인되었습니다. 이제 구독을 진행하실 수 있습니다.
```

### 3. 로그인 상태별 버튼 텍스트 변경 (선택사항)

**현재**: "구독하기" (항상 동일)

**개선안**:
- 로그아웃 상태: "로그인하고 구독하기"
- 로그인 상태: "구독하기"
